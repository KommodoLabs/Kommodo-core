# Security Properties

# 1. Arithmetic Safety

All contracts use Solidity:

```solidity
pragma solidity =0.8.24;
```

Solidity ≥0.8 provides built-in overflow and underflow protection.

Example:

```solidity
uint x = a + b;
```

This operation **automatically reverts on overflow**, preventing arithmetic corruption without requiring SafeMath.

**Impact**

* prevents integer overflow exploits
* ensures safe financial calculations

---

# 2. Deterministic Token Ordering (Factory)

`KommodoFactory` enforces canonical token ordering:

```solidity
(address token0, address token1) = assetA < assetB
    ? (assetA, assetB)
    : (assetB, assetA);
```

This ensures:

* deterministic pair identity
* prevention of duplicate pools with reversed token inputs

The mapping is also written in both directions:

```solidity
kommodo[assetA][assetB][poolFee] = address(_kommodo);
kommodo[assetB][assetA][poolFee] = address(_kommodo);
```

This guarantees symmetric lookup and prevents duplicate deployments.

---

# 3. Pool Uniqueness Enforcement

Pool duplication is prevented with:

```solidity
require(
    kommodo[assetA][assetB][poolFee] == address(0),
    "create: existing pool"
);
```

Because the reverse mapping is also populated, the same pool cannot be created again with reversed parameters.

---

# 4. Uniswap V3 Fee Tier Validation

The factory verifies that the requested pool fee exists in the Uniswap V3 factory:

```solidity
int24 tickSpacing = IUniswapV3Factory(factory).feeAmountTickSpacing(poolFee);
require(tickSpacing != 0, "constructor: invalid poolFee");
```

This prevents deployment of Kommodo pools referencing unsupported fee tiers.

---

# 5. Zero Address Protection

Deployment parameters are validated:

```solidity
require(_factory != address(0));
require(token0 != address(0));
```

This ensures:

* valid dependency configuration
* prevention of unusable pools

---

# 6. Factory-Controlled Pool Deployment

All `Kommodo` pools are deployed through:

```solidity
new Kommodo(...)
```

inside the factory contract.

This ensures:

* consistent constructor parameters
* canonical token ordering
* protocol-controlled pool creation
